## Обзор

Данный проект содержит:
- **vtsh** - командная оболочка, аналогичная bash
- **ioloader** - нагрузчик подсистемы ввода-вывода
- **cpu-mat-mul** - нагрузчик вычислительной подсистемы (перемножение матриц)
- **cpu-mat-mul-threaded** - многопоточная версия CPU нагрузчика с поддержкой параллельных вычислений
- **ema-replace-int** - нагрузчик для поиска и замены целых чисел в файле

## Сборка проекта

### Требования

- Компилятор GCC или Clang
- CMake версии 3.12 или выше
- Python 3 (для запуска тестов)

### Сборка командной оболочки vtsh

```bash
cd lab/vtsh
mkdir -p build
cd build
cmake ..
cmake --build .
```

После успешной сборки исполняемый файл будет находиться в `build/bin/vtsh`.

### Сборка программ-нагрузчиков

#### IO Loader (ioloader)

```bash
cd lab/vtsh/lib
gcc -o ioloader ioloader.c -Wall -Wextra
```

#### CPU Matrix Multiplication (cpu-mat-mul)

```bash
cd lab/vtsh/lib
gcc -o cpu-mat-mul cpu-mat-mul.c -Wall -Wextra
```

#### CPU Matrix Multiplication (многопоточная версия, cpu-mat-mul-threaded)

```bash
cd lab/vtsh/lib
gcc -pthread -o cpu-mat-mul-threaded cpu-mat-mul-threaded.c -Wall -Wextra
```

#### EMA Replace Int (ema-replace-int)

```bash
cd lab/vtsh/lib
gcc -o ema-replace-int ema-replace-int.c -Wall -Wextra
```

**Примечание:** Для компиляции с оптимизацией используйте флаг `-O3`:
```bash
gcc -O3 -o ioloader ioloader.c -Wall -Wextra
gcc -O3 -o cpu-mat-mul cpu-mat-mul.c -Wall -Wextra
gcc -O3 -pthread -o cpu-mat-mul-threaded cpu-mat-mul-threaded.c -Wall -Wextra
```

## Командная оболочка vtsh

### Описание

vtsh - это простая командная оболочка, которая позволяет:
- Запускать программы с аргументами
- Выполнять команды последовательно через оператор `;`
- Отображать время выполнения команд
- Запускать вложенные оболочки
- Обрабатывать команды, читающие из stdin (например, `cat`)

### Как работает vtsh

1. **Запуск программ:** Использует системный вызов `vfork()` для создания дочернего процесса
2. **Измерение времени:** Использует `clock_gettime()` для измерения времени выполнения команды
3. **Последовательное выполнение:** Оператор `;` позволяет выполнять команды одну за другой
4. **Обработка ошибок:** При ошибке выполнения команды выводится сообщение "Command not found"

### Запуск vtsh

```bash
./build/bin/vtsh
```

После запуска вы увидите приглашение:
```
vtsh> 
```

### Основные команды

#### Простые команды

```bash
vtsh> echo hello
hello
Время выполнения команды - 0.001234

vtsh> ls -la
[вывод команды ls]
Время выполнения команды - 0.002345
```

#### Последовательное выполнение команд (оператор `;`)

```bash
vtsh> echo first; echo second; echo third
first
Время выполнения команды - 0.001111
second
Время выполнения команды - 0.001222
third
Время выполнения команды - 0.001333
```

**Важно:** Все команды выполняются последовательно, независимо от статуса предыдущей команды.

#### Команды, читающие из stdin

```bash
vtsh> cat
hello
world
hello
world
Время выполнения команды - 0.002456
```

После ввода `cat` введите данные (по одной строке), затем пустую строку для завершения.

#### Вложенные оболочки

```bash
vtsh> ./vtsh
vtsh> echo hi
hi
Время выполнения команды - 0.001111
vtsh> exit
vtsh> echo back
back
Время выполнения команды - 0.001222
```

Также можно использовать `./shell`, `shell` или `vtsh` для запуска новой оболочки.

#### Выход из оболочки

```bash
vtsh> exit
```

Или используйте `e`, `q`, `quit`.

### Особенности реализации

- **vfork():** Используется для создания дочерних процессов (вариант proc-vfork)
- **Время выполнения:** Выводится в stderr, поэтому не попадает в перенаправление stdout
- **Обработка ошибок:** При отсутствии команды выводится "Command not found"
- **История команд:** Команды сохраняются в файл `history.txt`

## Программы-нагрузчики

### 1. IO Loader (ioloader)

#### Описание

Программа для нагрузки подсистемы ввода-вывода. Может выполнять чтение или запись блоков данных в файл с различными режимами доступа.

#### Параметры запуска

Все параметры обязательны, кроме `--range`:

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--rw` | Режим: `read` (чтение) или `write` (запись) | `--rw write` |
| `--block_size` | Размер блока в байтах | `--block_size 4096` |
| `--block_count` | Количество блоков на повторение | `--block_count 1000` |
| `--file` | Путь к файлу | `--file test.bin` |
| `--range` | Диапазон в файле (формат: `start-end`). По умолчанию: весь файл | `--range 0-10485760` |
| `--direct` | Использовать O_DIRECT: `on` или `off` | `--direct off` |
| `--type` | Тип доступа: `sequence` (последовательный) или `random` (случайный) | `--type sequence` |
| `--repetitions` | Количество повторений | `--repetitions 10` |

#### Примеры использования

**Последовательная запись:**
```bash
./ioloader --rw write --block_size 4096 --block_count 1000 \
  --file test.bin --range 0-10485760 --direct off \
  --type sequence --repetitions 10
```

**Случайное чтение:**
```bash
./ioloader --rw read --block_size 4096 --block_count 500 \
  --file test.bin --range 0-5242880 --direct off \
  --type random --repetitions 5
```

**С O_DIRECT (требует выравнивания):**
```bash
./ioloader --rw write --block_size 4096 --block_count 100 \
  --file test.bin --range 0-1048576 --direct on \
  --type sequence --repetitions 1
```

#### Вывод программы

Программа выводит:
- Параметры запуска
- Прогресс выполнения (для каждого повторения)
- Результаты:
  - Всего операций
  - Всего обработано байт (в байтах и MB)
  - Время выполнения
  - Пропускная способность (MB/сек)
  - IOPS (операций/сек)

#### Важные замечания

1. **O_DIRECT:** При использовании `--direct on`:
   - Размер блока должен быть кратен размеру блока файловой системы
   - Начало диапазона должно быть выровнено
   - Буфер должен быть выровнен (программа делает это автоматически)

2. **Размер файла:** Убедитесь, что файл достаточно большой для указанного диапазона и количества блоков.

3. **Производительность:**
   - Последовательный доступ (`sequence`) значительно быстрее случайного (`random`)
   - При последовательном доступе используется кэш ОС
   - При случайном доступе каждое обращение идет к диску

### 2. CPU Matrix Multiplication (cpu-mat-mul)

#### Описание

Программа для нагрузки вычислительной подсистемы. Выполняет перемножение матриц заданного размера.

#### Параметры запуска

Обязательные параметры:
- `--width1` - ширина первой матрицы
- `--height1` - высота первой матрицы
- `--width2` - ширина второй матрицы
- `--height2` - высота второй матрицы

Опциональные параметры:
- `--seed` - начальное значение для генератора случайных чисел (по умолчанию: 52)
- `--in_file_1` - файл с первой матрицей (если не указан, генерируется случайно)
- `--in_file_2` - файл со второй матрицей (если не указан, генерируется случайно)
- `--out_file` - файл для сохранения результата (по умолчанию: `mat-mul-output.txt`)

**Примечание:** Параметр `--repetitions` не поддерживается в текущей версии. Для многократного выполнения используйте многопоточную версию или запускайте программу несколько раз.

#### Формат параметров

Параметры можно передавать двумя способами:

1. Через знак равенства:
```bash
--width1=100 --height1=100 --width2=100 --height2=100
```

2. Отдельным аргументом:
```bash
--width1 100 --height1 100 --width2 100 --height2 100
```

#### Примеры использования

**Базовое использование:**
```bash
./cpu-mat-mul --width1=100 --height1=100 --width2=100 --height2=100
```

**С заданным seed:**
```bash
./cpu-mat-mul --width1=100 --height1=100 --width2=100 --height2=100 \
  --seed=123
```

**Многопоточная версия:**
```bash
./cpu-mat-mul-threaded --width1=500 --height1=500 --width2=500 --height2=500 \
  --threads=8
```

**С входными файлами:**
```bash
./cpu-mat-mul --width1=10 --height1=10 --width2=10 --height2=10 \
  --in_file_1=matrix1.txt --in_file_2=matrix2.txt \
  --out_file=result.txt
```

#### Формат файлов матриц

Если вы хотите использовать свои матрицы, формат файла:
```
<высота> <ширина>
<элемент_0_0> <элемент_0_1> ... <элемент_0_<ширина-1>>
<элемент_1_0> <элемент_1_1> ... <элемент_1_<ширина-1>>
...
<элемент_<высота-1>_0> <элемент_<высота-1>_1> ... <элемент_<высота-1>_<ширина-1>>
```

Пример:
```
3 3
1 2 3
4 5 6
7 8 9
```

#### Важные замечания

1. **Совместимость размеров:** `width1` должно равняться `height2` для возможности умножения.

2. **Сложность:** Алгоритм имеет сложность O(n³), где n - размер матрицы. Время выполнения растет кубически с размером.

3. **Память:** Убедитесь, что у вас достаточно памяти для матриц. Для матрицы 1000x1000 требуется ~8 MB памяти (для int64_t).

4. **Производительность:** При компиляции с `-O3` производительность может увеличиться в 1.6-2 раза.

### 2.1. CPU Matrix Multiplication (многопоточная версия, cpu-mat-mul-threaded)

#### Описание

Многопоточная версия CPU нагрузчика, использующая библиотеку `pthread` для параллельного выполнения вычислений на нескольких ядрах процессора. Матрица разбивается на строки, каждая из которых обрабатывается отдельным потоком.

#### Параметры запуска

Все параметры из однопоточной версии, плюс:
- `--threads` - количество потоков для параллельного выполнения (по умолчанию: 1)

#### Примеры использования

**Базовое использование (1 поток):**
```bash
./cpu-mat-mul-threaded --width1=500 --height1=500 --width2=500 --height2=500 --threads=1
```

**Использование всех ядер (8 потоков):**
```bash
./cpu-mat-mul-threaded --width1=500 --height1=500 --width2=500 --height2=500 --threads=8
```

**Оптимальное количество потоков (4 потока):**
```bash
./cpu-mat-mul-threaded --width1=500 --height1=500 --width2=500 --height2=500 --threads=4
```

#### Важные замечания

1. **Производительность:** Многопоточная версия может дать ускорение в 3-4 раза при использовании всех ядер процессора
2. **Оптимальное количество потоков:** Рекомендуется использовать количество потоков, равное количеству ядер процессора
3. **Сравнение с многопроцессным подходом:** Многопоточная версия эффективнее для одного экземпляра программы, но при запуске нескольких программ может возникнуть конкуренция за ресурсы
4. **Компиляция:** Требуется флаг `-pthread` при компиляции: `gcc -pthread -o cpu-mat-mul-threaded cpu-mat-mul-threaded.c`

### 3. EMA Replace Int (ema-replace-int)

#### Описание

Программа для поиска целых чисел в файле и замены их на другие значения. Работает с внешней памятью (файлами).

#### Параметры запуска

Позиционные аргументы (в порядке следования):

1. `[файл]` - имя файла (по умолчанию: `test_file.bin`)
2. `[повторений]` - количество повторений (по умолчанию: 1)
3. `[блоков]` - количество блоков на повторение (по умолчанию: 100)
4. `[random|sequential]` - тип доступа (по умолчанию: `random`)
5. `[размер_файла]` - размер файла в байтах (по умолчанию: 1 MB)
6. `[target_value]` - значение для поиска (-1 = случайное, по умолчанию: -1)
7. `[replacement_value]` - значение для замены (-1 = target_value + 1000, по умолчанию: -1)
8. `[init_random]` - инициализировать файл случайными значениями (`1` или `random` = да, `0` или `zero` = нет, по умолчанию: `0`)

#### Примеры использования

**Базовое использование:**
```bash
./ema-replace-int test.bin 1 1000 random 1048576
```

**С заданным целевым значением:**
```bash
./ema-replace-int test.bin 10 1000 random 1048576 500 1500
```

**С инициализацией случайными значениями:**
```bash
./ema-replace-int test.bin 10 1000 random 1048576 -1 -1 1
```

**Последовательный доступ:**
```bash
./ema-replace-int test.bin 5 500 sequential 5242880 -1 -1 1
```

#### Как работает программа

1. **Инициализация файла:**
   - Если файл не существует или меньше указанного размера, он создается
   - Если `init_random=1`, файл заполняется случайными значениями (0-1999)
   - Если `init_random=0`, файл заполняется нулями

2. **Поиск и замена:**
   - Программа читает целые числа (4 байта) из файла
   - Если значение совпадает с `target_value`, оно заменяется на `replacement_value`
   - Доступ может быть последовательным или случайным

3. **Вывод результатов:**
   - Количество операций замены
   - Количество найденных совпадений
   - Объем обработанных данных
   - Время выполнения
   - Пропускная способность и IOPS

#### Важные замечания

1. **Инициализация:** Для эффективного поиска рекомендуется использовать `init_random=1`, чтобы файл содержал случайные значения.

2. **Целевое значение:** Если `target_value=-1`, значение генерируется случайно при каждом запуске.

3. **Производительность:** Случайный доступ значительно медленнее последовательного из-за операций поиска на диске.

4. **Размер файла:** Убедитесь, что размер файла достаточен для количества операций.

## Часто задаваемые вопросы

### Q: Почему последовательный доступ быстрее случайного?

A: При последовательном доступе операционная система эффективно использует кэш страниц. Данные читаются/записываются последовательно, что позволяет предсказать следующие обращения. При случайном доступе каждое обращение требует поиска нужного сектора на диске, что создает значительные задержки.

### Q: Что такое O_DIRECT?

A: O_DIRECT - это флаг, который позволяет обходить кэш операционной системы и работать напрямую с диском. Это требует специального выравнивания буферов и может снизить производительность, но дает более точные измерения реальной скорости диска.

### Q: Почему время выполнения в time больше, чем внутреннее время программы?

A: Время в `time` включает накладные расходы на запуск программы, загрузку библиотек, инициализацию и т.д. Внутреннее время программы измеряет только время выполнения основной логики.

### Q: Как увеличить время выполнения тестов?

A: Увеличьте параметры:
- Для IO Loader: увеличьте `--block_count` и `--repetitions`
- Для CPU Matrix Mul: увеличьте размер матриц (`--width1`, `--height1`, etc.)
- Для EMA Replace Int: увеличьте размер файла и `--repetitions`
- Для многопоточного CPU: используйте большие матрицы (500x500 или больше)

### Q: Можно ли использовать нагрузчики параллельно?

A: Да, вы можете запустить несколько экземпляров в фоне:

**IO Loader:**
```bash
./ioloader --rw write --block_size 4096 --block_count 1000 \
  --file test1.bin --range 0-10485760 --direct off \
  --type sequence --repetitions 100 &
./ioloader --rw write --block_size 4096 --block_count 1000 \
  --file test2.bin --range 0-10485760 --direct off \
  --type sequence --repetitions 100 &
wait
```

**CPU Matrix Mul (многопроцессный):**
```bash
for i in 1 2 3 4; do
  ./cpu-mat-mul --width1=500 --height1=500 --width2=500 --height2=500 > /dev/null 2>&1 &
done
wait
```

**CPU Matrix Mul (многопоточный):**
```bash
./cpu-mat-mul-threaded --width1=500 --height1=500 --width2=500 --height2=500 --threads=8
```

### Q: В чем разница между многопроцессным и многопоточным подходом?

A: 
- **Многопроцессный подход:** Каждый процесс имеет свою собственную память, что требует больше ресурсов, но обеспечивает изоляцию
- **Многопоточный подход:** Потоки разделяют память процесса, что более эффективно, но требует синхронизации
- Для CPU нагрузчика многопоточный подход обычно эффективнее для одного экземпляра программы
- Оптимальное количество процессов для системы с 8 ядрами - 4 процесса
- Оптимальное количество потоков - равно количеству ядер процессора (8 потоков)

### Q: Насколько улучшается производительность с оптимизацией -O3?

A: Компиляция с `-O3` дает ускорение примерно в 1.6-1.7 раза для CPU нагрузчика:
- Real time сокращается на ~40%
- User time сокращается на ~43%
- System time практически не изменяется
- Оптимизация улучшает использование процессора через векторизацию и оптимизацию циклов
